# THE BUTTON — 全人類が押す1つのボタン

全世界の人が同じ1つのカウンターを押し続けるゲーム。
スタミナ・連続ログイン・レアクリティカル・破壊の駆け引きで、
希少性と中毒性を最大化した「押すだけ」のバカゲー。合言葉でグループも作れる。

## ゲームルール

### 押す
- スタミナを1消費して押す。世界カウントが増える
- 押すたびに低確率でレア発動：
  - 通常 +1
  - ⚡ クリティカル（5%）+10
  - 💥 スーパークリティカル（0.5%）+100
  - 🎰 ジャックポット（0.05%）+1000（全員に通知）

### 希少性（行動制限）
- **スタミナ**：最大10。使い切ると一定時間押せない。1分で1回復
- スタミナ制で「連打して放置」を防ぎ、こまめに戻ってくる設計

### 中毒性（戻る理由）
- **連続ログインボーナス**：日付をまたいで来るたびに連続日数アップ。連続日数ぶん最大スタミナが増える（上限+20）。途切れるとリセット
- **称号**：連続日数で「新人→常連→中毒者→指の達人→神の指→押下の鬼→伝説」

### 破壊（コスト制）
- 破壊ゲージ（=自分の累計貢献）が `100 × 現在レベル` に達すると世界を0に戻せる
- 高レベルほど破壊が重い＝タダ壊し封じ
- 破壊神の殿堂に名前と回数が記録される。peak記録は不滅

### レベル
- 世界カウントが閾値（1,000→5,000→1万→5万→10万…）を超えるとレベルアップ、全員に演出

## 仕組み
- フロント：静的HTML 1枚（`index.html`）→ GitHub Pages
- バックエンド：Firebase Realtime Database（無料枠・サーバー管理不要）
- 同期：リアルタイム。カウントは `runTransaction` で競合なし
- ※スタミナ・連続ログインはブラウザのlocalStorage管理（ゆる制限。データ消去で突破可能）

## セットアップ

### 1. firebaseConfig を貼る
Firebaseコンソール → プロジェクト設定（歯車）→「マイアプリ」→ ウェブアプリ `</>` → `firebaseConfig` を `index.html` の該当箇所に貼り替え。

```js
const firebaseConfig = {
  apiKey: "...",
  authDomain: "one-button-7ab53.firebaseapp.com",
  databaseURL: "https://one-button-7ab53-default-rtdb.firebaseio.com",
  projectId: "one-button-7ab53",
};
```

### 2. データベースのルール
Realtime Database →「ルール」タブに `database.rules.json` の中身を貼って公開。

### 3. GitHub Pages
`git push` 後、公開URL：`https://chiyoko-san.github.io/one-button/`

## 調整ポイント（index.html上部の定数）
- `STAMINA_REGEN_MS` … スタミナ回復間隔（既定60秒）
- `BASE_MAX_STAMINA` … 基本最大スタミナ（既定10）
- `rollPress()` … レア確率の調整

## さらなる拡張アイデア
